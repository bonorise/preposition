# about 介词页设计

日期：2026-03-21
状态：已确认

## 1. 背景

当前站点的介词详情页已经采用统一骨架：顶部释义与关键提示、3D 图解、例句分区、相关词、对比、常用组合、易错点、测验与 FAQ。新增 `about` 时，用户明确要求参考其他介词页面结构，不额外发明新页面模块，并优先用轻松、有趣、形象的方式把词义讲明白；SEO 与 GEO 作为第二优先级，在不破坏教学体验的前提下自然覆盖。

`about` 的内容难点在于：它最常见的学习入口是“关于 / 大约”，但这两个义项背后其实都能追溯到“围着一个中心转”的原始感觉。当前站点的分类切换只有 `space / time / dynamic`，没有 `abstract`。因此本次设计不扩展新分类，而是用现有结构把“空间图像 -> 时间近似 -> 抽象延伸”串起来。

## 2. 决策摘要

- 页面类型：继续使用现有普通介词详情页，不新增页面专属模块
- 内容重心：教学优先，强调画面感和记忆钩子；SEO/GEO 自然嵌入
- 释义策略：在最开始的 `meaning` 中直接写入“围着某人或某物转圈”的核心解释
- 分类策略：仍只使用现有 `space / time / dynamic`
- 抽象义策略：不新增 `abstract` 分类，把“关于、对于、因为、为了、忙于、大约”等延伸义放进释义、tips、例句、对比、FAQ 中
- 图解策略：用 `dynamic` 演示“绕圈”，用 `space` 演示“在周围 / 到处”，用 `time` 演示“在某个时间点或数字附近”

## 3. 页面内容设计

### 3.1 顶部释义

`about` 的首屏释义不先堆字典义，而先给用户一个能看见的感觉。推荐文案方向：

`about：原始感觉像围着一个人或事物转圈。于是它既可以表示在周围、围绕着，也能引申成关于某事，还能表示大约、左右，好像在某个时间点或数字附近打转。`

这样处理有三个目的：

1. 用户一进入页面就能抓住核心图像，而不是被散乱义项分散注意力。
2. “关于”与“大约”第一次在同一个解释链条里出现，利于建立迁移记忆。
3. 首屏自然覆盖 `about meaning / about preposition / about vs around / about = approximately / about = concerning` 等检索意图，但读感仍然像教学内容。

### 3.2 关键提示

保留 3 条提示，全部服务于“同一个核心感觉的不同延伸”：

1. 围着具体东西转：在周围、在附近、在各处、围绕
2. 围着话题转：关于、对于、涉及
3. 围着数字或时间点转：大约、左右、差不多

提示语气应尽量口语化，避免写成词典条目。

### 3.3 例句组织

例句仍放在现有例句区里，但内容顺序要从具体到抽象：

- `space`：围着某物、在某处到处、在附近
- `time`：about ten o’clock / about twenty people / about ten minutes
- 抽象义：talk about, worry about, happy about, a book about space, what is this movie about?

不扩展新的例句分类；抽象义例句优先作为基础例句和对比、FAQ 的支撑材料出现。

## 4. 3D 图解设计

### 4.1 空间场景（space）

目标不是表现“紧贴某物”，而是表现“围着中心展开 / 在某处各处可见”。画面应让用户感觉球体并非固定在一侧，而是围着中心对象活动或分布。

推荐表现：

- 基础球位放在中心对象外侧，不强调接触
- 使用能体现“周围”的布局，而不是 `on / beside / under` 那种单方向关系
- 文案同步解释：`about` 在空间义里像“围着它转、在它周围活动”

### 4.2 时间场景（time）

使用现有时间轴组件，表现“about 10 o’clock”这类“在一个点附近”的感觉。视觉上不应像精确命中某个刻度，而要更像围绕该点的近似范围。

推荐表现：

- 将 `about` 纳入时间介词分类
- 时间轴中心仍以一个点为锚，但球的位置略偏于精确刻度，强化“左右、大约”
- 配合例句说明：不是精确十点，而是十点前后

### 4.3 动态场景（dynamic）

这是 `about` 的记忆核心场景，用来承接“原始感觉是绕圈”。使用现有路径动画让球体围着中心对象运动，作为首要视觉钩子。

推荐表现：

- 使用闭环路径动画
- 路径比 `around` 更强调“这个动作是词义来源”，不是单纯“绕着走一圈”的动作描述
- 让用户点击 `Play` 时能直观看到“about 的老底子是转圈”

## 5. 对比、易错与问答设计

### 5.1 主对比：about vs around

这是页面最重要的对比组，必须明确：

- 两者都能表示“在周围 / 大约”
- `around` 更偏具体空间绕圈或在某处周围移动
- `about` 更容易进入抽象话题、情绪原因、涉及方面等用法
- 数字与时间里两者都能用，但 `about` 更适合作为教学入口，帮助解释“关于”和“大约”的共同来源

### 5.2 次对比：about vs on

轻量提示即可，重点放在“内容围绕某主题”：

- `a book about art`
- `a lecture on art`

不展开学术化讨论，只帮助初学者理解 `about` 的“围着一个主题打转”。

### 5.3 常见错误

优先覆盖以下误区：

- 只把 `about` 背成“关于”，看不出它和“大约”有共同核心
- 看不懂 `be worried about / happy about / angry about` 中 `about` 指向的是“围绕某事产生情绪”
- 把 `discuss about` 当成标准搭配

### 5.4 FAQ

优先回答：

1. `about` 为什么既能表示“关于”又能表示“大约”？
2. `about` 和 `around` 有什么区别？
3. 为什么 `about` 后面常接时间、数字、人、事和情绪原因？

## 6. SEO / GEO 设计

本页不走“堆关键词”路线，而是通过可摘录、可引用的短句自然覆盖检索意图。核心策略：

- 在 `meaning` 中用一句话讲清中心义
- 在 `comparison` 与 `FAQ` 中覆盖问答型检索
- 在例句中同时覆盖空间、时间和抽象义

适合被引用的核心句应当简短稳定，例如：

- `about 的核心感觉是围着一个中心打转。`
- `所以它既能表示在周围，也能引申成关于和大约。`

这类句子既适合零基础用户快速理解，也更容易被搜索摘要或 AI 回答抽取。

## 7. 实现影响范围

本次不需要改详情页架构，主要是数据与少量分类、SEO 配置扩展：

- `src/data/prepositions.ts`
  - 新增 `about` 条目
  - 补齐 `meaning / tips / examples / examplesByCategory / comparison / collocationGroups / commonMistakes / quiz / faq / scenesByCategory`
  - 补充 `about` 的 `relatedIds` 映射
- `src/lib/prepositionCategory.ts`
  - 将 `about` 纳入时间介词分类
- `src/app/[locale]/p/[id]/page.tsx`
  - 如需更强 SEO/GEO 落点，为 `about` 添加定制 metadata override
- `public/thumbnails/*` 与 `scripts/generate-thumbnails.ts`
  - 新增词条后补生成 `about` 的缩略图资源

## 8. 验收标准

以下全部满足，视为 `about` 词条页符合本次设计：

1. 首页和详情页可以正常访问 `/[locale]/p/about`
2. 首屏释义第一段直接讲清“about 的原始感觉像围着中心打转”
3. 详情页可以切换 `space / time / dynamic`
4. `dynamic` 场景能直观看到“绕圈”动画
5. `time` 场景能表达“大约、左右”的时间点近似感
6. 文案覆盖“在周围 / 关于 / 大约 / 情绪原因 / 涉及方面”等核心延伸义
7. 页面保留站内统一结构，不新增专属模块
8. 对比、易错点、FAQ 能回答 `about vs around` 与“为什么 about 能表示关于和大约”这两个核心问题
